Sales Management System

I created a Sales Management System for my term project in my Component Based Software Design course with my goals aimed at streamlining sales operations for a fictional company. Through a combination of database management, programming in C#, and the creation of user-friendly interfaces using Windows Forms, the system is designed to enhance productivity and organization within sales teams. Key components include the management of client information, scheduling of appointments, and tracking of performance metrics. By emphasizing user-friendly interfaces and intuitive functionalities the project aims to provide a seamless experience for users while effectively managing sales-related tasks. Overall, this initiative represents a practical application of theoretical concepts in a real-world context.

Temple Capstone Project Storage Platform

Myself and three other classmates were tasked with developing a comprehensive application for Temple University's CIS Department to effectively manage and track projects undertaken in Capstone 1-2 courses at Temple University. Currently, the CIS Department relies on platforms like Google Drive, GitHub, and Canvas, which lack integration and organization between eachother. The system seeks to streamline project documentation, code storage, and project status tracking. System goals include categorizing projects by status, semester, and team, implementing role-based access controls, and facilitating audit logging. Users, including students and administrators, will have tailored permissions for project management. Expected benefits include improved project efficiency, organization, and collaboration. The project timeline spans analysis, design, build, and implementation phases across two semesters.
